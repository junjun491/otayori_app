# API通信設計（ローカル / 本番）

本プロジェクトでは、ローカルと本番で責務を分離しています。

## ローカル開発

```
Browser
  ↓
Next.js（3000）
  ↓ rewrites
Rails API（3001）
```

### ポイント

- 同一オリジンで通信可能
- CORS不要
- 開発体験を簡潔に

## 本番環境

```
Browser
  ↓
ALB
  ↓
Rails API
```

### ポイント

- ALB が単一入口
- パスベースルーティング
- Next.js は API に関与しない

## なぜこの構成か

- ローカル：開発しやすさ重視
- 本番：シンプルで運用しやすい構成

## 意識したこと

- 開発体験と本番構成の整合性
- 責務の明確化
